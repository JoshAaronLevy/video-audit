# Premiere Bridge Shared Contract

This file documents the MVP filesystem bridge contract shared by the Express backend and the Premiere Pro UXP plugin. The executable constants and lightweight validators live in `premiereBridge.cjs`.

## Constants

| Name | Value |
| --- | --- |
| Plugin ID | `video-audit-premiere-bridge` |
| Default bridge directory | `~/VideoAudit/premiere-bridge` |
| Default export output directory | `/Users/joshlevy/Movies/Edited` |
| Premiere project bin name | `Video Audit Exports` |
| Request type | `export-selected-videos` |
| Heartbeat max age | `30000` ms |
| Max videos per request | `100` |

## Bridge Layout

```txt
~/VideoAudit/premiere-bridge/
  status.json
  requests/
  completed/
  failed/
  presets/
```

The backend creates and writes request JSON files under `requests/`. The UXP plugin reads those files, processes each selected video individually, and then moves the request payload with result metadata to `completed/` or `failed/`.

## Presets

Preset metadata is stable and shared. The actual Adobe `.epr` file should live in the bridge `presets/` folder.

```json
[
  {
    "id": "h264-1080p-12mbps",
    "label": "H.264 1080p - 12 Mbps",
    "resolution": "1920x1080",
    "presetFileName": "h264-1080p-12mbps.epr",
    "available": true
  }
]
```

The UI should display the friendly `label`, but it should send only the stable `id` to the backend. `GET /api/premiere/status` returns this preset metadata with runtime file availability. If the backing `.epr` file is missing or unreadable, `available` is `false` and the object may include `unavailableReason` and `unavailableMessage`.

## Request JSON

The backend writes one request file per export submission. A request may include multiple videos, but the plugin should create one sequence and one Adobe Media Encoder queue item per video.

```json
{
  "id": "8f65f05f-7a7c-41f0-9e98-8c865c6af811",
  "type": "export-selected-videos",
  "status": "queued",
  "createdAt": "2026-05-11T00:00:00.000Z",
  "presetId": "h264-1080p-12mbps",
  "presetFileName": "h264-1080p-12mbps.epr",
  "outputDirectory": "/Users/joshlevy/Movies/Edited",
  "videos": [
    {
      "id": "/Users/example/Videos/example.mp4",
      "fileName": "example.mp4",
      "absolutePath": "/Users/example/Videos/example.mp4",
      "directory": "/Users/example/Videos",
      "durationSeconds": 123.45,
      "width": 3840,
      "height": 2160,
      "displayAspectRatio": "16:9",
      "frameRate": 29.97
    }
  ]
}
```

Required fields:

- `id`: UUID generated by the backend.
- `type`: `export-selected-videos`.
- `status`: one of `queued`, `processing`, `completed`, or `failed`.
- `createdAt`: ISO timestamp.
- `presetId`: known preset ID.
- `presetFileName`: preset file name mapped from `presetId`.
- `outputDirectory`: `/Users/joshlevy/Movies/Edited`.
- `videos`: one to 100 selected videos.

Each video requires `id`, `fileName`, `absolutePath`, and `directory`. Metadata fields such as `durationSeconds`, `width`, `height`, and `frameRate` may be numbers or `null`; `displayAspectRatio` may be a string or `null`.

## Status JSON

The UXP plugin writes `status.json` as a heartbeat. The backend reads it during `GET /api/premiere/status`.

```json
{
  "plugin": "video-audit-premiere-bridge",
  "status": "ready",
  "updatedAt": "2026-05-11T00:00:00.000Z",
  "activeProjectName": "Some Project.prproj",
  "activeProjectPath": "/Users/example/Projects/Some Project.prproj",
  "bridgeDir": "/Users/example/VideoAudit/premiere-bridge",
  "outputDirectory": "/Users/joshlevy/Movies/Edited",
  "version": "0.1.0"
}
```

Status values:

- `ready`: Premiere is open, the plugin has bridge/output folder access, and an active project is available.
- `not_ready`: the plugin is loaded but missing required access or project state.
- `error`: the plugin hit a recoverable bridge error and should expose a message in its panel/status file.

## Completed Request JSON

When a request succeeds, the plugin moves the request to `completed/<id>.json` and appends result fields.

```json
{
  "id": "8f65f05f-7a7c-41f0-9e98-8c865c6af811",
  "type": "export-selected-videos",
  "status": "completed",
  "completedAt": "2026-05-11T00:05:00.000Z",
  "presetId": "h264-1080p-12mbps",
  "presetFileName": "h264-1080p-12mbps.epr",
  "outputDirectory": "/Users/joshlevy/Movies/Edited",
  "queuedJobs": [
    {
      "sourcePath": "/Users/example/Videos/example.mp4",
      "sequenceName": "Video Audit - example",
      "outputPath": "/Users/joshlevy/Movies/Edited/example-1080p.mp4"
    }
  ]
}
```

## Failed Request JSON

When a request fails, the plugin moves the request to `failed/<id>.json` and appends failure details.

```json
{
  "id": "8f65f05f-7a7c-41f0-9e98-8c865c6af811",
  "type": "export-selected-videos",
  "status": "failed",
  "failedAt": "2026-05-11T00:05:00.000Z",
  "error": {
    "message": "Unable to import selected video into Premiere."
  }
}
```

Keep failure details concise. The bridge files are a local handoff format, not a long-term job database.
